# 廟埕守香 專案文件

## 專案概述

《廟埕守香》是一款以台灣神明信仰為主題的文字塔防遊戲。玩家在廟埕棋盤上請出法器與神名字，阻止妖魔鬼怪沿路徑逼近香爐。遊戲的核心特色是「中文字」本身就是單位，並透過合成、神明組合、部首攻擊動畫與台灣廟宇視覺，形成辨識度。

目前專案是純前端靜態網頁，不需要後端服務。

## 核心玩法

玩家使用香火請令，抽出一般法器或神名字，拖曳到棋盤可放置格上鎮守。

- 目標：守住香爐，撐過 12 波妖魔。
- 資源：香火。
- 抽取：按下「請令」消耗香火。
- 放置：將下方法壇中的字拖曳到棋盤。
- 合成：相同字、相同等級可以合成升級。
- 丟棄：將字拖到「化去」圖示，或選取後按「化去」，可回收部分香火。
- 波次：完成佈陣後按「迎第 N 波」開戰，戰鬥中可暫停或繼續。
- 倍速：按下「1×／2×」可切換戰鬥速度，出怪、移動、攻擊與冷卻會同步加速。
- 聲音：第一次操作後開始播放廟埕五聲音階，標題列的音符按鈕可切換音樂與音效。
- 活動：每日簽到、每日任務與每日活動會記錄進度並提供香火獎勵。
- 敗北：妖魔抵達香爐會扣香爐耐久，耐久歸零失敗。

## 操作方式

- 拖曳法壇中的字到棋盤：出戰。
- 拖曳棋盤上的字到其他可放置格：移動。
- 拖曳相同等級的同字：合成升級。
- 拖曳字到「化去」：丟棄並回收香火。
- 點選字後按「化去」：丟棄選取字。
- 點選「迎第 N 波」：開始下一波。
- 點選「暫停／繼續」：控制目前波次。
- 點選「1×／2×」：切換正常速度與兩倍速度。
- 點選「重開」：重新開始。

## 一般法器

一般法器是 `符、鏡、鈴、劍、印`。一般法器最多升到 5 級。

| 字 | 名稱 | 定位 | 攻擊方式 |
| --- | --- | --- | --- |
| 符 | 鎮煞符 | 大範圍攻擊 | 符陣掃向多個敵人，不暈、不慢 |
| 鏡 | 照妖法鏡 | 遠距多目標照射 | 鏡光照妖，照射多個目標 |
| 鈴 | 淨壇法鈴 | 緩速牽制 | 鈴聲震盪，使妖魔短時間變慢 |
| 劍 | 斬煞法劍 | 近距高傷 | 對單體造成高傷，附斬擊效果 |
| 印 | 敕令法印 | 鎮壓控制 | 對單體造成傷害並暈眩 |

## 神名字與組合

神名字會隨機請出，不由一般法器升級而來。神名字單獨放置時不啟動攻擊，必須與對應另一字相鄰才會成為神明組合。

| 組合 | 神明 | 定位 | 攻擊方式 |
| --- | --- | --- | --- |
| 媽 + 祖 | 媽祖 | 遠距穿透 | 對多個目標穿透攻擊 |
| 關 + 公 | 關公 | 近距斬擊 | 對範圍內目標斬擊 |
| 城 + 隍 | 城隍爺 | 鎮壓控制 | 傷害並暈眩敵人 |
| 觀 + 音 | 觀音 | 遠距護香 | 攻擊多個目標，累積出手後恢復香爐耐久 |
| 王 + 爺 | 王爺 | 巡擊 | 對多個目標巡行攻擊 |

神明組合規則：

- 兩字相鄰才會啟動。
- 啟動後才可以繼續升級。
- 神明組合共用一個等級，最高 5 級。
- `媽` 與 `祖` 本身沒有攻擊力，組成 `媽祖` 後才會攻擊與升級。
- `觀` 只能隨機出現，不會由其他字升級而來。

## 敵人

敵人以妖魔鬼怪字形呈現，不使用黑色圓圈。

| 字 | 名稱 | 特性 |
| --- | --- | --- |
| 怪 | 路煞小怪 | 第一波起出現的基礎敵人 |
| 鬼 | 孤魂野鬼 | 第二波起出現，較耐打、較慢 |
| 妖 | 狐魅妖邪 | 第三波起出現，移動較快 |
| 魈 | 山魈鬼王 | 第 4 波魔王，負傷過半後會疾走，抵達香爐扣 2 點 |
| 煞 | 赤煞鬼將 | 第 8 波魔王，護甲最高且較難控制，抵達香爐扣 3 點 |
| 魔 | 黑煞魔王 | 第 12 波最終魔王，高血量、強抗控制，抵達香爐扣 5 點 |

敵人會有小型血條，血條位於字上方。被鈴影響時，走路動畫變慢；被印或城隍爺鎮壓時，才會暈眩。

大魔王使用更大的完整字形與加寬血條，不使用黑色圓圈、光暈或全畫面震動。魔王會受到鈴與鎮壓效果，但控制時間會依魔王抗性縮短。

## 升級與化去

### 合成升級

- 一般法器：同字、同等級可合成，最高 5 級。
- 神名字：同字、同等級可合成，但必須位於已啟動神明組合的格子上。
- 神名字啟動後兩字共用等級，最高 5 級。

### 化去回收

化去會刪除字並回收香火。

- 一般法器回收較少。
- 神名字回收較多。
- 等級越高，回收越多。
- 棋盤上的字化去時，會在原位置顯示 `+香火`。

## 美術與介面方向

目前視覺方向是台灣廟埕、廟宇、法壇、朱印、香火。

已使用圖片素材：

- `assets/temple-backdrop.webp`：廟埕背景。
- `assets/temple-board.webp`：棋盤與面板質感。
- `assets/discard-burner.webp`：化去圖示。
- `assets/nav-battle.png`：底部「守香」分類圖示。
- `assets/nav-checkin.png`：底部「簽到」分類圖示。
- `assets/nav-tasks.png`：底部「任務」分類圖示。
- `assets/nav-events.png`：底部「活動」分類圖示。
- `assets/nav-codex.png`：底部「神明譜」分類圖示。

底部分類使用獨立透明圖片，不在遊戲中顯示產圖用的合併圖或來源圖。選取分類時只改變按鈕層次，不替文字或圖示加模糊光暈。

設計注意事項：

- 字要清楚，不使用會讓字糊掉的光暈。
- 棋盤不震動，避免暈眩。
- 出戰字下方不加額外色條。
- 出戰的特殊字不加白色方塊底。
- 攻擊時才做部首拆動，平常不拆小字。
- 法壇中的字不要動。
- 連結框應像朱印，不要像 debug 紅框。

## 音樂與音效

聲音由瀏覽器即時合成，不依賴網路或外部音檔。

- 背景音樂：低音量五聲音階與低頻香爐音景。
- 操作音效：請令、神名字出現、落位、合成、化去、暫停、繼續與加持。
- 戰鬥音效：符、鏡、鈴、劍、印各有不同音色，神明組合另有降臨與攻擊音。
- 戰況音效：敵兵消散、魔王進場與降伏、香爐受擊、波次完成、勝利與失敗。
- 音訊會在玩家第一次操作後啟動，符合瀏覽器自動播放限制。
- 靜音選擇會保留在瀏覽器中。

## 每日系統

每日資料以玩家裝置日期為準，在新的一天第一次開啟遊戲時重置；簽到與領獎狀態保存在瀏覽器中。

### 每日簽到

- 七日循環獎勵，連續簽到會依序提高香火。
- 中斷一天以上會從第 1 日重新開始。
- 第 7 日領取後，下一日開始新的七日循環。

### 每日任務

- 壇前請令：請令 3 次。
- 法器相合：合成 1 次。
- 守住香路：完成 2 波。
- 驅除邪祟：降伏 20 名敵兵。

### 每日活動

- 驅煞巡境：降伏 30 名妖邪。
- 魔王懸賞：降伏 1 名大魔王。
- 三巡淨路：完成 3 波守香。

任務完成後必須在活動中心手動領取香火。

### 次要介面

- 點選底部分類後，只打開該分類介面，不在介面內重複顯示其他活動分頁。
- 每日簽到：顯示今日天數、今日獎勵、七日獎勵表與重置時間。
- 每日任務：顯示總完成度、各任務進度與一鍵領取。
- 每日活動：顯示巡境完成度、魔王波次與一鍵領取。
- 神明譜：列出法器、神明合字、攻擊力、射程、攻擊特色與妖邪資料。
- 內容超過手機高度時只在介面內捲動，主戰鬥畫面保持原位。

### `index.html`

頁面結構與主要介面：

- 標題列
- 戰況狀態
- 棋盤
- 法壇
- 神明譜
- 加持選擇視窗
- 結果視窗

負責整體視覺、棋盤、法壇、字體、敵人、血條、攻擊特效、部首動畫、化去圖示與神明組合框。

### `js/`

拆分後的遊戲邏輯：

- `config.js`：棋盤大小、路徑、法器、神名字、敵人、加持資料。
- `state.js`：DOM 參照、遊戲狀態、拖曳狀態。
- `audio.js`：背景音樂、遊戲音效、音訊節流與靜音偏好。
- `activities.js`：每日簽到、任務、活動進度、領獎、儲存與活動中心切換。
- `render.js`：棋盤、法壇、敵人、HUD、神明組合框渲染。
- `input.js`：點選、拖曳、放置、合成、化去回收。
- `flow.js`：請令、波次、敵人生成與移動、法器攻擊、神明組合攻擊。
- `effects.js`：飛字、部首、斬擊、鎮字、光束、浮字等視覺特效。
- `app.js`：勝敗、重開、事件綁定與遊戲啟動。

## 開發與 AI 準則 (Development & Agent Rules)

1. **純文字視覺表現**：對戰畫面嚴禁使用圖示/圖片取代文字，實體必須以漢字與 CSS 動態表現。
2. **字體大小限制**：最小字體不可低於 `0.85rem` (14px)。
3. **神明字元與法器顏色**：同一神明的所有字元（如「媽」與「祖」）必須統一顏色，嚴禁針對單一字元覆寫顏色。所有基礎法器（`符`、`鏡`、`鈴`、`劍`、`印`）文字統一為古墨深色 (`#17110c`)。
4. **跨平台字體與動態**：全平台優先載入 `Noto Serif TC` 與 `LXGW WenKai TC`，並保留偏旁拆解、斬煞法劍出鞘大弧度斬擊 (`artifactSwordPartSlash`) 與墨影動態。
5. **APP 退背景暫停**：APP/網頁進入背景或螢幕鎖定時，必須將 Web Audio API `AudioContext` 掛起暫停音樂。
6. **Android 返回鍵保護**：返回鍵優先關閉彈窗 Modal / Hub，無視窗時才提示退出 App。
